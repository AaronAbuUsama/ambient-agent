# The application owns one logging root

Ambient Agent creates a single Pino root logger and every runtime voice flows through it: `whatsappd` receives a child logger through its public `SessionConfig.logger` seam instead of creating its own, Effect's logger is bridged into another child, and direct console calls in application code are replaced. The root applies credential redaction before any transport sees a line, so tokens, OAuth records, and webhook secrets cannot leak through a component that logs its own configuration.

On a TTY the root renders a flat, semantic operator feed with human-readable timestamps and one line per event. Managed-chat inbound text and confirmed Ambience Say text are emitted as info-level chat records to the console sink; message text is not debug-only. Normal output also shows accepted Window processing, deliberate silent settlement, otherwise-private non-empty final output, completion duration, retries, and failures. It omits Pino level labels, subsystem names, internal correlation ids, tool activity, thinking activity, and nested JSON. Ignored chats, history replay, self-message echoes, and lower-level diagnostics remain behind `--debug`.

In every mode the root also writes the full structured Pino records, including those info-level chat records, to size-capped rotating JSON files under `~/.ambient-agent/logs/`, which answers "what happened last night on the VPS" without depending on a service manager's journal discipline. The console sink renders the chat records as human-readable lines; the file sink retains their structured fields. Credentials are still redacted at the root. Remote crash telemetry such as Sentry is explicitly out of scope: local logs are the diagnostic story, and any future telemetry would be a separate opt-in decision, never a substitute for this module.
