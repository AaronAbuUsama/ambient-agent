# Managed data lives in ~/.ambient-agent on every platform

The managed data directory is `~/.ambient-agent` on macOS, Linux, and any future platform, replacing the platform-native defaults (XDG data home, Application Support). Ambient Agent is a consumer CLI product, and its owner must be able to find its data from memory the way they find `~/.ssh` or `~/.claude`; the July 2026 VPS proof showed the platform-native default failing exactly that test. The explicit `dataDirectory` override and its absolute-path validation remain the configuration escape hatch, and `init` and `status` always print the resolved directory so the location is never inferred.

An existing installation at the former platform-native path is adopted exactly once: when `~/.ambient-agent` does not exist and the legacy directory does, startup moves it atomically (same-filesystem rename, staging-directory fallback) before any component opens a database, and records the migration in the application database. A machine with both directories present fails closed with both paths named rather than choosing one silently. Directory and credential permission requirements (0700/0600) are unchanged from ADR 0011.
